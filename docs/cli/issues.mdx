# Issues

The CLI provides four commands for working with issues: `list`, `show`, `create`, and `comment`.

## looped list

List issues with optional filters and pagination.

```bash
looped list [options]
```

### Options

| Flag | Description | Default |
|------|-------------|---------|
| `--status <status>` | Filter by status (`triage`, `todo`, `backlog`, `in_progress`, `done`, `canceled`) | all |
| `--type <type>` | Filter by issue type (`signal`, `hypothesis`, `plan`, `task`, `monitor`) | all |
| `--project <id>` | Filter by project ID | none |
| `--priority <n>` | Filter by priority (0-4) | all |
| `--limit <n>` | Results per page | `50` |
| `--offset <n>` | Pagination offset | `0` |

### Examples

List all issues:

```bash
looped list
```

Filter by status:

```bash
looped list --status todo
```

Filter by type and project:

```bash
looped list --type task --project clx1abc2def3ghi4jkl5mnop
```

Output as JSON for scripting:

```bash
looped list --status in_progress --json | jq '.[].title'
```

## looped show

Show full detail for a single issue, including labels, relations, children, and agent results.

```bash
looped show <id>
```

### Arguments

| Argument | Description |
|----------|-------------|
| `id` | Issue ID (CUID2) |

### Example

```bash
looped show clx1abc2def3ghi4jkl5mnop
```

## looped create

Create a new issue. In a terminal (TTY), the CLI prompts interactively for type, priority, and project if flags are not provided. In non-TTY environments, defaults are used.

```bash
looped create <title> [options]
```

### Arguments

| Argument | Description |
|----------|-------------|
| `title` | Issue title (required) |

### Options

| Flag | Default (non-TTY) | Description |
|------|-------------------|-------------|
| `--type <type>` | `task` | Issue type (`signal`, `hypothesis`, `plan`, `task`, `monitor`) |
| `--priority <n>` | `3` (medium) | Priority (0=none, 1=urgent, 2=high, 3=medium, 4=low) |
| `--project <id>` | none | Project ID |
| `--description <text>` | none | Issue description |
| `--parent <id>` | none | Parent issue ID |

### Examples

Create with all flags (scriptable):

```bash
looped create "Fix login timeout" \
  --type task \
  --priority 2 \
  --description "Login requests timing out after 30s" \
  --project clx1abc2def3ghi4jkl5mnop
```

Create interactively (terminal prompts for type, priority, project):

```bash
looped create "Investigate checkout error spike"
```

Create a child issue:

```bash
looped create "Add cache warming" --parent clx1abc2def3ghi4jkl5mnop
```

### Sample output

```
Created #45 [ğŸ”§ task] Fix login timeout
```

## looped comment

Add a comment to an issue.

```bash
looped comment <issueId> <body>
```

### Arguments

| Argument | Description |
|----------|-------------|
| `issueId` | Issue ID (CUID2) |
| `body` | Comment text |

### Example

```bash
looped comment clx1abc2def3ghi4jkl5mnop "Confirmed this is a cache issue"
```

### Sample output

```
Comment added to issue clx1abc2def3ghi4jkl5mnop
```

Comments are attributed to `looped-cli` with `authorType: human`.
